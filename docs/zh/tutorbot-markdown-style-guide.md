# TutorBot Markdown 风格指南

本指南用于约束 TutorBot 在微信小程序等移动端场景下的答案输出，目标是：

- 结构稳定
- 视觉简洁
- 对 LLM 输出波动不敏感
- 能被当前渲染链路稳定消费

## 推荐结构

优先使用 3 到 5 个稳定区块：

```md
## 结论
一句话直接给判断。

## 判断依据
- 依据 1
- 依据 2

## 踩分点
1. **点 1：** 说明
2. **点 2：** 说明

## 易错点
- **易错点 1：** 说明
- **易错点 2：** 说明
```

## 允许的 Markdown 子集

- 二级或三级标题
- 普通段落
- 单层无序列表
- 单层有序列表
- 加粗
- 简单引用块
- 必要时的简单表格

## 推荐写法

- 小标签统一写成 `- **标签：** 内容`
- 有序点统一写成 `1. **标签：** 内容`
- 箭头强调统一写成 `文本 → **结论**`
- 段落和区块之间只保留一个空行

## 不推荐写法

- 多层嵌套列表
- 在某个 bullet 下再缩进 2 层继续列点
- 一行里混太多强调、箭头、括号和长句
- 用 HTML 直接控制样式
- 把正文包在代码块里

## 例子

### 推荐

```md
## 踩分点
1. **时间限制：** 必须记住 24 小时。
2. **顺序要求：** 初拧、复拧、终拧都要在 24 小时内完成。

## 易错点
- **不要混淆：** 不是 48 小时，也不是 72 小时。
```

### 不推荐

```md
1. **时间限制**：必须记住 24 小时
   - 这是考试常考点
   - 还要注意不是 48 小时
```
